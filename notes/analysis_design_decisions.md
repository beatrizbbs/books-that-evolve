# Analysis Design Decisions

This note records two decisions after the cleaning phase:

1. How to build project-specific theme buckets.
2. How to handle duplicate/repeated works before trend analysis.

## Theme Buckets

Decision: use **multi-label theme buckets**, not one exclusive theme per book.

Reason: the project asks how dominant themes changed over time, and Goodreads books often belong to overlapping waves. A single book can legitimately be `Dystopian`, `Young Adult`, `Romance`, and `Science Fiction`. Forcing one label would hide the cultural overlap this project is trying to study.

Theme tags should be built from:

- `genres_clean`
- `title_clean`
- `description_clean`

Recommended first-pass buckets:

- `middle_grade_fantasy`
- `mythology_adventure`
- `paranormal_supernatural`
- `paranormal_romance`
- `dystopian_post_apocalyptic`
- `epic_high_fantasy`
- `romantasy_fae`
- `contemporary_realistic`
- `science_fiction`
- `horror_dark`
- `graphic_manga_comics`

Use Goodreads genres as the strongest signal, then title/description keywords as supporting evidence.

Examples:

- `dystopian_post_apocalyptic`: genres or text include `Dystopia`, `Dystopian`, `Post Apocalyptic`, `apocalypse`, `survival`, `rebellion`, `district`, `regime`.
- `paranormal_romance`: genres include both paranormal/supernatural signals and romance signals, or text includes terms like `vampire`, `werewolf`, `angel`, `demon`, `mate`, `forbidden love`.
- `romantasy_fae`: text or genres include `fae`, `faerie`, `fairy`, `court`, `kingdom`, `shadow`, `curse`, plus romance/fantasy signals.
- `middle_grade_fantasy`: genres include `Middle Grade`, `Childrens`, or youth labels plus fantasy/magic/adventure signals.
- `mythology_adventure`: text or genres include `mythology`, `gods`, `Greek`, `Roman`, `Norse`, `quest`, `prophecy`, `demigod`.

Recommended output format:

- Keep boolean columns such as `theme_dystopian_post_apocalyptic`.
- Also create `theme_count` and possibly a list column like `theme_tags`.
- For charts, count a book in every bucket it matches.
- For share-of-attention charts, use fractional weighting if needed: each book contributes `popularity_score / theme_count` across its matched themes.

Implemented outputs:

- `data/processed/analysis/books_themed.csv`
- `data/processed/bridge/book_themes.csv`
- `data/processed/reference/theme_summary.csv`

Theme buckets are generated by:

```bash
.venv/bin/python scripts/build_theme_features.py
```

## Deduplication

Decision: use **two layers**:

1. Remove exact repeated records for the main cleaned analysis.
2. Keep edition-like or title-author-year duplicates flagged unless a specific chart needs canonical works.

Current duplicate findings in the 1997-2024 project scope:

- Rows in scope: 37,327
- Duplicate `bookId` rows: 76 rows across 38 repeated IDs
- Duplicate `title_key` + `author_key` rows: 807 rows
- Duplicate `title_key` + `author_key` + `first_publish_year` rows: 660 rows
- Dropping duplicate `bookId` rows would remove only 38 rows.
- Dropping duplicate `title_key` + `author_key` + `first_publish_year` rows would remove 562 rows.

Recommendation:

- For the default analysis dataset, dedupe by `bookId`, keeping the row with the highest `numRatings_clean`, then highest `bbeVotes_clean`, then highest `averageRating`.
- Do not automatically dedupe by `title_key` + `author_key`, because those groups include translations, editions, omnibus editions, volume splits, and legitimate reissues.
- Keep `duplicate_title_author_key` and `duplicate_clean_isbn` as flags.
- For “era-defining books” tables, show one canonical row per work using `title_key` + `author_key` + `first_publish_year`, keeping the highest `numRatings_clean`.
- For “blockbuster effect” analysis, keep individual books, not series collapsed into one row, because the README defines the unit of analysis as individual books.

Practical default:

```python
books = pd.read_csv("data/processed/core/books_cleaned.csv")
books_scope = books[books["in_project_scope_1997_2024"]].copy()

books_main = (
    books_scope
    .sort_values(["numRatings_clean", "bbeVotes_clean", "averageRating"], ascending=False)
    .drop_duplicates(["bookId"], keep="first")
    .copy()
)
```

Optional canonical-work view:

```python
books_work_level = (
    books_main
    .sort_values(["numRatings_clean", "bbeVotes_clean", "averageRating"], ascending=False)
    .drop_duplicates(["title_key", "author_key", "first_publish_year"], keep="first")
    .copy()
)
```

Use `books_main` for trend analysis. Use `books_work_level` only for tables where repeated editions would look messy.

`bbeScore` decision: keep raw `bbeScore` only for traceability. Do not use it in the default analysis or popularity score because the metric is opaque and largely redundant with `bbeVotes`.
